# CCCP Advisory System

Локальная RAG-система (Retrieval-Augmented Generation) с версионированием базы знаний. Система принимает `.md`-файлы в виде `.zip`-архивов, хранит их в MinIO, автоматически индексирует в векторной базе данных Qdrant и отвечает на вопросы пользователей, используя только информацию из активной версии базы знаний.

## Быстрый старт

```bash
# Запуск системы
docker compose up --build -d

# Загрузка базы знаний (автоматически индексируется)
curl -X POST http://localhost/api/v1/knowledge-base/versions \
  -F "archive=@knowledge.zip" -F "comment=Initial version"

# Активация версии
curl -X POST http://localhost/api/v1/knowledge-base/versions/1/activate

# Задать вопрос
curl -X POST http://localhost/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "Какие продукты вы предлагаете?"}'
```

При первом запуске Ollama скачает необходимые модели (~5 ГБ). Дождитесь сообщения `All models are ready.` в логах контейнера `ollama`, после чего система готова к работе.

## Сервисы

| Сервис   | Порт      | Назначение                                    |
|----------|-----------|-----------------------------------------------|
| nginx    | 80        | Единая точка входа (reverse proxy)            |
| admin    | внутренний | Управление версиями базы знаний               |
| query    | внутренний | Ответы на вопросы пользователей               |
| postgres | 5432      | Метаданные версий (kb_versions, kb_files)     |
| minio    | 9000/9001 | S3-хранилище файлов базы знаний               |
| qdrant   | 6333      | Векторная база данных                         |
| ollama   | 11434     | Inference-сервер (embeddings + LLM)           |

Прямой доступ к admin и query закрыт — все запросы проходят через nginx.

## API

Все эндпоинты доступны через nginx на порту 80. Префикс: `/api/v1/knowledge-base`.

---

### POST /versions

Загрузка новой версии базы знаний из `.zip`-архива. Файлы сохраняются в MinIO и сразу индексируются в Qdrant (статус `ingested`).

**Запрос:**

```bash
curl -X POST http://localhost/api/v1/knowledge-base/versions \
  -F "archive=@knowledge.zip" \
  -F "comment=Добавлены новые продукты"
```

| Поле      | Тип    | Обязательное | Описание                      |
|-----------|--------|--------------|-------------------------------|
| `archive` | file   | да           | `.zip`-архив с `.md`-файлами  |
| `comment` | string | нет          | Комментарий к версии          |

**Ответ (201):**

```json
{
  "id": "319f8659-54be-4016-8771-10e93943d7e4",
  "version_num": 1,
  "created_at": "2026-04-12T14:06:48.126177Z",
  "status": "ingested",
  "s3_prefix": "versions/v1/",
  "qdrant_collection": "kb_v1",
  "file_count": 2,
  "comment": "Добавлены новые продукты",
  "files": [
    {"filename": "info.md", "s3_key": "versions/v1/info.md", "size_bytes": 865, "sha256": "..."},
    {"filename": "products.md", "s3_key": "versions/v1/products.md", "size_bytes": 1148, "sha256": "..."}
  ]
}
```

| Код | Причина                                              |
|-----|------------------------------------------------------|
| 400 | Файл не `.zip` или не содержит `.md`                |
| 500 | Ошибка при индексации (MinIO/Qdrant/Ollama недоступны) |

Список всех версий, отсортированных по номеру (новые первые).

```bash
curl http://localhost/api/v1/knowledge-base/versions
```

**Ответ (200):** массив объектов версий (формат как в POST /versions).

```bash
curl http://localhost/api/v1/knowledge-base/versions/active
```

**Ответ (200):** объект версии с `status: "active"`.

| 404 | Нет активной версии |

### GET /versions/active/archive

Скачать активную версию как `.zip`-архив.

```bash
curl -O http://localhost/api/v1/knowledge-base/versions/active/archive
```

**Ответ:** файл `kb_v{N}.zip` (`Content-Type: application/zip`).

| Код | Причина             |
|-----|---------------------|
| 404 | Нет активной версии |

---

### GET /versions/{version_num}/archive

Скачать конкретную версию как `.zip`-архив.

```bash
curl -O http://localhost/api/v1/knowledge-base/versions/1/archive
```

| Код | Причина           |
|-----|-------------------|
| 404 | Версия не найдена |

---

### POST /versions/{version_num}/activate

Активация версии: переключение Qdrant-alias `kb_active` на коллекцию данной версии. Предыдущая активная версия переводится в статус `ingested`.

```bash
curl -X POST http://localhost/api/v1/knowledge-base/versions/1/activate
```

**Ответ (200):** объект версии с `status: "active"`.

| Код | Причина                                  |
|-----|------------------------------------------|
| 404 | Версия не найдена                        |
| 409 | Версия не в статусе `ingested`           |

---

### DELETE /versions/{version_num}

Удаление версии и всех её данных: файлы из MinIO, коллекция из Qdrant, записи из PostgreSQL. Активную версию удалить нельзя — сначала активируйте другую.

```bash
curl -X DELETE http://localhost/api/v1/knowledge-base/versions/1
```

**Ответ:** `204 No Content`.

| Код | Причина                                            |
|-----|----------------------------------------------------|
| 404 | Версия не найдена                                  |
| 409 | Нельзя удалить активную версию                     |

---

### POST /ask

Принимает вопрос на естественном языке и возвращает ответ, сгенерированный на основе релевантных фрагментов из активной версии базы знаний.

**Запрос:**

```bash
curl -X POST http://localhost/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "Какие продукты вы предлагаете?"}'
```

| Поле       | Тип    | Обязательное | Описание          |
|------------|--------|--------------|-------------------|
| `question` | string | да           | Вопрос пользователя |

**Ответ (200):**

```json
{
  "answer": "Согласно информации из базы знаний, мы предлагаем..."
}
```

| Поле     | Тип    | Описание                                           |
|----------|--------|----------------------------------------------------|
| `answer` | string | Сгенерированный ответ или `"Информация не найдена"` |

| Код | Причина                                                        |
|-----|----------------------------------------------------------------|
| 400 | Проблемы с кодировкой запроса (не удалось декодировать тело)   |
| 500 | Qdrant или Ollama недоступны, ошибка при генерации ответа      |

Сервис автоматически пытается перекодировать тело запроса из windows-1251, cp866, koi8-r и iso-8859-5 в UTF-8, если исходная кодировка не UTF-8.

## Версионирование базы знаний

### Жизненный цикл версии

```
Upload (.zip) -> ingested -> Activate -> active
```

- **ingested** — файлы загружены в MinIO, эмбеддинги сгенерированы, коллекция `kb_v{N}` создана в Qdrant. Версия готова к активации.
- **active** — alias `kb_active` указывает на коллекцию этой версии. Все запросы `/ask` используют эту версию. Только одна версия может быть активной.
- **failed** — ошибка при индексации.

При активации новой версии предыдущая активная версия переводится в статус `ingested`. Коллекции старых версий не удаляются — возможен откат через повторную активацию. Активную версию нельзя удалить.

### Хранение в MinIO

```
knowledge-base/
  versions/
    v1/
      info.md
      products.md
    v2/
      info.md
      products.md
      faq.md
```

Каждая версия хранится в отдельном префиксе. Файлы старых версий не удаляются до явного удаления версии.

### Хранение в Qdrant

Каждая версия создаёт отдельную коллекцию `kb_v{N}`. Именованный alias `kb_active` всегда указывает на коллекцию активной версии. Query-сервис запрашивает только alias, не зная о номерах версий.

## Алгоритм RAG

RAG (Retrieval-Augmented Generation) — подход, при котором ответ языковой модели основывается не на её внутренних знаниях, а на конкретных документах, найденных по запросу. Это позволяет получать точные ответы, привязанные к актуальному содержимому базы знаний, и минимизирует галлюцинации модели.

Алгоритм состоит из двух фаз: **индексация** (при загрузке новой версии) и **поиск с генерацией** (при каждом вопросе).

### Фаза 1: Индексация (Ingestion)

Цель — преобразовать текстовые документы в векторные представления и сохранить их в базе данных для быстрого семантического поиска. Индексация выполняется автоматически при загрузке новой версии.

```
.zip архив -> MinIO -> Чанкинг -> Эмбеддинги -> Qdrant (kb_v{N})
```

#### Шаг 1. Загрузка файлов

Из `.zip`-архива извлекаются только `.md`-файлы (скрытые файлы и директории игнорируются). Файлы загружаются в MinIO под префиксом `versions/v{N}/`.

#### Шаг 2. Разбиение на чанки (Chunking)

Каждый документ разбивается на небольшие фрагменты (чанки) с помощью `RecursiveCharacterTextSplitter` из LangChain. Алгоритм разбиения:

- **Размер чанка:** 300 символов. Это достаточно мало, чтобы каждый чанк содержал одну смысловую единицу, и достаточно велико, чтобы сохранить контекст.
- **Перекрытие (overlap):** 100 символов. Соседние чанки перекрываются на 100 символов, чтобы информация, оказавшаяся на границе разбиения, не потерялась.
- **Рекурсивная стратегия:** сплиттер сначала пытается разделить текст по двойным переносам строк (`\n\n`), затем по одинарным (`\n`), затем по пробелам, и только в крайнем случае — посимвольно. Это сохраняет естественную структуру текста (абзацы, предложения).

#### Шаг 3. Генерация эмбеддингов

Каждый чанк преобразуется в числовой вектор (эмбеддинг) с помощью модели **BGE-M3**, запущенной в Ollama. BGE-M3 — мультиязычная модель, поддерживающая русский язык.

Эмбеддинг — это вектор фиксированной размерности, в котором семантически близкие тексты располагаются рядом друг с другом в векторном пространстве. Все чанки отправляются на генерацию эмбеддингов одним батчем.

#### Шаг 4. Сохранение в Qdrant

Создаётся новая коллекция `kb_v{N}` с параметрами:

- **Размер вектора:** определяется автоматически по размерности первого эмбеддинга.
- **Метрика расстояния:** косинусное сходство (Cosine).

Каждый чанк сохраняется как точка (point) с:
- **id** — порядковый номер чанка (0, 1, 2, ...).
- **vector** — эмбеддинг чанка.
- **payload** — `{"filename": "...", "text": "..."}` — имя файла и исходный текст чанка.

### Фаза 2: Поиск и генерация ответа (Retrieval + Generation)

Цель — найти наиболее релевантные фрагменты базы знаний и сгенерировать ответ на их основе.

```
Вопрос -> Эмбеддинг -> Поиск в kb_active -> Контекст + Промпт -> LLM -> Ответ
```

#### Шаг 1. Эмбеддинг вопроса

Вопрос пользователя преобразуется в вектор той же моделью **BGE-M3**, что использовалась при индексации. Это критически важно: поиск работает корректно только если вопрос и документы закодированы одной и той же моделью.

#### Шаг 2. Семантический поиск

Вектор вопроса сравнивается со всеми векторами чанков в Qdrant (alias `kb_active`). Поиск выполняется по косинусному сходству (от 0 до 1, где 1 — полное совпадение).

Параметры поиска:
- **top_k = 3** — возвращаются не более 3 наиболее похожих чанков.
- **similarity_threshold = 0.3** — чанки с косинусным сходством ниже 0.3 отбрасываются как нерелевантные.

Если ни один чанк не прошёл порог, система сразу возвращает `"Информация не найдена"`, не обращаясь к LLM.

#### Шаг 3. Формирование промпта

Тексты найденных чанков объединяются в один блок контекста (разделитель — двойной перенос строки). Затем формируются два сообщения для LLM:

**Системный промпт** задаёт роль и ограничения:
> Ты — полезный ИИ-ассистент. Отвечай на вопросы пользователя, используя ТОЛЬКО предоставленный контекст. Если в контексте недостаточно информации для ответа, ответь строго: "Информация не найдена".

**Пользовательское сообщение** содержит контекст и вопрос:
```
Контекст:
{тексты найденных чанков}

Вопрос: {вопрос пользователя}

Ответь на вопрос, используя только информацию из контекста выше.
```

#### Шаг 4. Генерация ответа

Промпт отправляется в модель **Qwen 2.5 7B**, запущенную в Ollama. Модель генерирует ответ, основываясь исключительно на предоставленном контексте.

### Схема полного цикла

```
                     ЗАГРУЗКА И ИНДЕКСАЦИЯ (автоматически при upload)
                     ================================================

  .zip архив (POST /versions)
         |
         v
  +------------------+
  | Извлечение .md   |
  | файлов из архива |
  +------------------+
         |
         v
  +------------------+
  | MinIO            |     prefix: versions/v{N}/
  | (сохранение)     |
  +------------------+
         |
         v
  +------------------+
  | PostgreSQL       |     kb_versions + kb_files
  | (метаданные)     |
  +------------------+
         |
         v
  +------------------+     chunk_size=300
  | Чанкинг          |     chunk_overlap=100
  +------------------+
         |
         v
  +------------------+
  | Эмбеддинги       |     модель: BGE-M3 (Ollama)
  +------------------+
         |
         v
  +------------------+
  | Qdrant           |     коллекция: kb_v{N}
  | (upsert)         |     status: ingested
  +------------------+
         |
         v  (POST /versions/{N}/activate)
  +------------------+
  | Alias            |     kb_active -> kb_v{N}
  | (переключение)   |     status: active
  +------------------+


                      ПОИСК + ГЕНЕРАЦИЯ (на каждый запрос)
                      ====================================

  Вопрос пользователя (POST /ask)
         |
         v
  +------------------+
  | Эмбеддинг        |     модель: BGE-M3 (Ollama)
  | вопроса          |
  +------------------+
         |
         v
  +------------------+     top_k=3
  | Поиск в Qdrant   |     similarity_threshold=0.3
  | (alias kb_active)|
  +------------------+
         |
     найдено?
    /        \
  нет        да
   |          |
   v          v
  "Информация  +------------------+
  не найдена"  | Формирование     |
               | промпта          |
               +------------------+
                        |
                        v
               +------------------+
               | Генерация        |     модель: Qwen 2.5 7B (Ollama)
               | ответа (LLM)    |
               +------------------+
                        |
                        v
                  Ответ пользователю
```

## Используемые модели

| Модель      | Назначение | Описание                                                             |
|-------------|------------|----------------------------------------------------------------------|
| BGE-M3      | Embeddings | Мультиязычная модель эмбеддингов с поддержкой русского языка         |
| Qwen 2.5 7B | LLM        | Языковая модель для генерации ответов на основе контекста            |

Обе модели запускаются локально через Ollama и скачиваются автоматически при первом старте.

## Управление базой знаний

### Добавление новой версии

1. Подготовьте `.zip`-архив с `.md`-файлами.
2. Загрузите: `curl -X POST http://localhost/api/v1/knowledge-base/versions -F "archive=@kb.zip"` (автоматически индексируется).
3. Активируйте: `curl -X POST http://localhost/api/v1/knowledge-base/versions/{N}/activate`

### Откат на предыдущую версию

```bash
curl -X POST http://localhost/api/v1/knowledge-base/versions/{N}/activate
```

### Удаление версии

```bash
curl -X DELETE http://localhost/api/v1/knowledge-base/versions/{N}
```

Активную версию удалить нельзя — сначала активируйте другую.

### Импорт из legacy-директории

```bash
docker compose exec admin uv run python scripts/seed_kb_v1.py
curl -X POST http://localhost/api/v1/knowledge-base/versions/1/activate
```
